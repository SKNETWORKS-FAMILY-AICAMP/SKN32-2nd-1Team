2. 프로젝트 개요

2-1. 프로젝트 배경 및 문제 정의

알뜰폰(MVNO)의 공세와 시장 패러다임 변화

합리적 소비 트렌드 확산과 자급제 단말 보급의 증가로 인해 가성비를 추구하는 젊은 세대를 중심으로 알뜰폰(MVNO) 시장이 유례없이 빠른 속도로 성장하고 있습니다. 이로 인해 기존 통신 3사(SKT, KT, LGU+)의 번호이동 가입자 이탈 위험이 급증하고 있습니다.

통신사 마케팅 비용 지출의 딜레마

가입자 기반 요금 수익 사업의 한계상, 신규 가입자 확보를 위한 보조금 경쟁(출혈 경쟁)은 피할 수 없으나 이는 곧 기업 영업이익의 급격한 악화를 수반합니다.

데이터 기반 '타겟 리텐션(Target Retention)'의 필요성

마케팅 경제학 원리에 따르면 신규 고객을 유치하는 비용(CAC)은 기존 고객을 유지하는 비용(CRC)보다 평균 5배 이상 비쌉니다. 따라서 이탈 위험이 높은 고위험군 고객층을 미리 정밀하게 식별해 내어 결합 상품 제안 등 맞춤형 케어를 제공하는 것이 마케팅 예산 대비 극적인 효과를 발휘합니다.

2-2. 프로젝트 목표

15단계 실험을 거친 최적의 예측 파이프라인 구축: 로지스틱 회귀 베이스라인에서 시작하여 데이터 증강, SMOTE 오버샘플링, 트리 기반 앙상블 모델 최적화 과정을 거쳐 최종적으로 목표 성능 수준을 만족하는 Churn 예측 시스템을 완성합니다.

비즈니스 가치에 집중된 평가지표 최적화: 단순 정확도(Accuracy)의 착시 현상을 경계하고, 실제 이탈할 고객을 성공적으로 찾아내는 재현율(Recall)과 모델의 변별력을 검증하는 ROC-AUC 지표를 최상으로 확보합니다.

사용자 행동 시그널의 과학적 증명: 통신사 고객들이 해지 전 남기는 월평균 이용 금액, 결합 상품 여부, 소득 수준 등의 상관관계를 정량적 가중치로 밝혀냅니다.

지능형 보안 로그인 및 실무용 데모 웹앱 배포: 실무 상담원 및 마케터가 사내 고객 데이터를 다룰 수 있도록 얼굴 2차 보안 인증(InsightFace)을 탑재하고, 직관적인 Churn 예측 결과 카드를 제공하는 Streamlit UI를 배포합니다.

2-3. 비즈니스적 기대 효과

출혈 경쟁 중단 및 예산 세이브: 이탈 고위험 고객층에만 가치 제안 및 보조금 혜택을 정밀 투여함으로써 연간 수십억 원에 달하는 마케팅 예산의 가성비 극대화.

안정적인 매출 파이프라인 확보: Churn Rate를 능동적으로 통제하여 이동통신 비즈니스의 근원적 자산인 '누적 가입자 기반 매출(MRR)'을 견고하게 수성.

보안 위협 대응: 고객 개인정보를 다루는 마케팅 부서 특성에 맞추어 안면 인증 기술을 선제 적용해 사내 기밀 유출 리스크 방지.

3. 주요 기능

🔐 1) 얼굴 인식 기반 지능형 2차 보안 로그인 (Multi-Factor Auth)

1차 인증: 사용자 데이터베이스(MySQL)와 연동된 아이디 및 패스워드 비밀번호 해시 검증.

2차 안면 인증: 내장/외장 카메라를 통한 실시간 사용자 얼굴 캡처 후, InsightFace와 OpenCV를 사용하여 기등록된 안면 벡터 임베딩 정보와의 유사도를 판단($\text{Similarity Threshold} \ge 0.40$).

보안 적용: 기업 내부 마케팅 대시보드의 개인정보 보호 체계를 강화하기 위한 아키텍처 구현.

📡 2) 머신러닝 기반 선제적 Churn 예측 진단

실시간 예측 피드백: 나이, 성별, 학력, 소득수준, 거주지, 가입 기간, 월 이용금액, 태블릿/웨어러블 기기 보유 여부 등의 인구통계학적/통신 이용 데이터를 입력 폼에 기입하는 즉시 이탈 확률(%) 산출.

위험 등급 자동 분류: 계산된 이탈 확률을 기반으로 직관적인 색상의 신호등 가이드 카드 출력.

🟢 안정 ($0\% \le \text{Probability} < 40\%$)

🟡 주의 관찰 ($40\% \le \text{Probability} < 60\%$)

🚨 이탈 위험 ($60\% \le \text{Probability} \le 100\%$)

📊 3) 실무 최적화 의사결정 시각화 대시보드

AI 리텐션 가이드: 이탈 예측 결과에 따라 맞춤형 대응 가이드라인을 자동 제안 (예: "장기 결합 할인 갱신 제안 대상자", "요금제 하향 조정 상담 권유" 등).

통신사별 통계 뷰: XGBoost 단일 모델과 전처리 파이프라인 모델 탭 구성을 통해 각 모델별 특징을 비교 분석할 수 있는 시각화 차트 스페이스 제공.

5. 데이터 전처리 결과서 (EDA)

5-1. 분석용 데이터셋 스펙 정의

출처: 미디어통계포털(KISDI) 한국미디어패널조사 원시 다개년 패널 데이터 (2015년 ~ 2024년 결합)

표본 정제 로직: 연도별 응답자 추적이 가능하도록 설계된 종단면 데이터의 특성을 고려하여, 설문 누적 응답 횟수가 최소 10회 이상인 고신뢰성 가입자 표본만 한정 수집.

종합 전처리 규모: 정제 완료 데이터 총 31,572건 (이 중 잔류 19,837건, 이탈 11,735건)

5-2. 전처리 파이프라인 핵심 흐름

타겟 변수(Target) 정의: 전년도($T$) 이용 통신사와 당해 연도($T+1$) 가입 통신사를 완전 비교하여, 통신사 브랜드가 다르게 변경된 시점을 이탈(1)로 정의하고 유지 시(0)로 정의하여 Churn 여부 산출.

결측치 및 극단치 정제: 결측치 표기(.) 및 응답 거부 코드(9999)를 도메인 지식에 근거하여 해당 연도의 평균값 혹은 최빈값으로 대치 적용.

원-핫 인코딩: 거주 지역(region), 혼인 여부(mar), 가입 통신사(telecom) 등의 명목형 변수를 One-Hot 형태로 변환하여 알고리즘 연산 가독성 제어.

5-3. 주요 EDA 인사이트 요약

결합 상품 가입 여부의 락인(Lock-in) 효과: mobile_bundle(결합 여부 = 1)에 가입되어 있는 가입자 그룹은 연간 이탈율이 미가입 그룹에 비해 약 40% 이상 하락하는 견고한 잠금 효과를 검증했습니다.

소득 대비 요금 부담률의 연동성: 소득 지표 대비 요금 청구 금액 비중을 나타내는 임시 가공 변수의 변동 폭이 임계 한계치를 초과할 때, 이탈 예측 확률이 기하급수적으로 폭증함을 확인했습니다.

9. 설치 및 실행 가이드

🛠️ 개발 요구 환경

OS: Windows 10/11 또는 macOS Ventura 이상

Runtime: Python 3.10 또는 Python 3.11 설치 권장

Database: MySQL Server 8.0 이상 로컬 구동 필요

1) 가상환경 생성 및 실행 환경 전입

# 가상환경 구축
python -m venv .venv

# 가상환경 활성화 (Windows)
.venv\Scripts\activate

# 가상환경 활성화 (macOS / Linux)
source .venv/bin/activate


2) 핵심 의존성 설치

pip install -r requirements.txt


💡 Windows 환경에서 dlib 빌드 실패 오류 시 긴급 대응책:
Windows에서는 C++ 컴파일 빌더 꼬임으로 안면 인식용 패키지 설치 시 dlib 에러가 날 수 있습니다. 이 경우 아래 바이너리 설치 방법을 적용해 주세요.

pip uninstall dlib face_recognition -y
pip install dlib-bin
pip install face_recognition --no-deps


3) MySQL 로컬 패스워드 주입

프로젝트 보안 구성을 위해 app/db.py 내부의 데이터베이스 접속 비밀번호를 본인의 MySQL 접속 비밀번호와 일치시킵니다.

# app/db.py
DB_PASSWORD = os.getenv("MYSQL_PASSWORD", "본인의_로컬_MySQL_패스워드_입력")


4) Streamlit 로컬 웹 서비스 실행

streamlit run app.py


명령어가 정상 작동하면 브라우저에 http://localhost:8501 주소의 메인 애플리케이션 대시보드가 자동으로 로드됩니다.

10. 개발자 테스트 모드 및 사용 시나리오

⚙️ DEV_SKIP_LOGIN 개발자 모드 이용 팁 (app.py)

데모 시연 촬영 전, 매번 웹캠 카메라 인식을 통한 로그인 인증 단계가 번거롭다면 메인 파일 상단의 스위치를 켜 편리하게 스킵할 수 있습니다.

# app.py 36라인
DEV_SKIP_LOGIN = True   # 복잡한 얼굴 인증 절차 없이 즉시 메인 예측 진단 탭으로 진입
DEV_SKIP_LOGIN = False  # [발표 및 최종 제출 모드] 실제 얼굴 촬영 및 2차 로그인 활성화


👤 실무자 가상 사용 가이드

회원 가입 및 얼굴 등록: ID/PW 입력 및 웹캠 카메라 위젯을 바라보고 촬영하여 안면 특징점 임베딩 데이터를 DB에 최초 1회 탑재합니다.

2차 얼굴 로그인: 패스워드 인증을 진행한 뒤, 카메라를 이용해 2차 안면 유사도를 산출하여 안전하게 시스템 로그인을 완료합니다.

이탈 위험 진단: 예측하고자 하는 고객 정보를 입력한 후 "이탈 여부 예측하기" 버튼을 눌러 게이지 스코어를 바탕으로 타겟 방어 솔루션을 수립합니다.

11. 프로젝트 한계점 및 팀원 회고

11-1. 프로젝트 한계점

가공 설문 패널 데이터의 한계성: 통신사 현업의 실제 로그 데이터(접속 차단 횟수, 실시간 패킷량, 소액결제 기록 등)가 아닌, 미디어 정보 포털의 정형 설문 조사 결과이다 보니 변수 특성의 동적인 변동 폭이 부드러워 예측 경계선 구축에 데이터 도메인적인 한계가 존재했습니다.

이탈(Churn)의 주관적 정의: 명확한 탈퇴 기록 데이터가 포함되어 있지 않아 직전 연도와의 가입 통신사 변동 유무를 팀원들이 직접 코딩으로 산식 가공하여 타겟 변수로 전환하였기에, 비즈니스 실제 상황과 완전히 밀치시키기 어려운 가정이 내포되어 있습니다.
